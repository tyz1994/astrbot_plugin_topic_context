# 织忆

> 织忆你太美。遗忘是时间的慈悲，Bot，你的Bot不需要慈悲。

基于主题的记忆组织与上下文管理，为 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 提供跨会话长期记忆能力。

## 设计理念

- **架构简单清晰** — 没有复杂的依赖和抽象，主题组织、纯文本存储，Markdown + JSON，看得懂、改得动
- **轻量透明** — 记忆数据就是普通的文件，你可以随时打开查看、编辑、备份，不被任何格式锁定
- **用户可控** — 通过 Web 管理界面自由增删改查主题与记忆片段，用户能掌控的记忆才是好记忆

## 安装

将本仓库克隆到 AstrBot 的 `data/plugins/` 目录下。

## 使用说明

### 1. 配置 Provider

在 AstrBot 管理面板的插件配置页面，为「记忆总结」和「主题匹配」分别配置独立的 LLM Provider。推荐使用**轻量、非思考模型**（如 GPT-4o-mini、Claude Haiku 等），主题匹配和记忆总结不需要深度推理，轻量模型响应更快、成本更低。

### 2. 回补历史记忆（可选）

如果是首次安装或从其他记忆插件迁移，可以使用冷启动命令扫描历史对话来构建初始记忆：

```
/memory coldstart 7
```

扫描过去 7 天的历史对话，自动总结、按主题归类并写入记忆。天数范围 1-365，根据实际需要调整。冷启动期间不影响正常使用。

### 3. 开始聊天

配置完成后无需其他操作，插件会在后台自动工作：

- 每轮对话后自动总结内容并归类到对应主题
- 用户发消息时自动匹配相关主题，将记忆注入上下文
- Bot 可通过工具调用主动检索更早的记忆细节
- 从用户的负反馈中提炼经验，避免同类错误

你也可以通过管理命令和 Web 界面随时查看、管理记忆：

| 命令 | 说明 |
|------|------|
| `/memory topics` | 查看所有主题 |
| `/memory show <主题名>` | 查看指定主题的记忆详情 |
| `/memory coldstart [天数]` | 扫描历史对话构建初始记忆 |


## 功能

### 对话总结

自动总结对话摘要并按主题存储。每个主题维护一份**核心记忆**，追踪该主题的最新动态；同时从负反馈中沉淀**经验记忆**，帮助 Bot 在同类场景中做出更好的回应。记忆碎片会定期合并整理，保持记忆库的精简与准确。

### 上下文注入

根据用户发送的消息自动匹配相关主题，将该主题的核心记忆和经验注入上下文，让 Bot 在对话中自然地延续之前的认知，无需用户重复交代背景。

### Agent 自主检索

依靠 Agent 能力自主挖掘非高频使用的深度记忆，在需要时通过 function-calling 按需查阅记忆片段，补充细节，弥补自动注入覆盖不到的长尾场景。

### 冷启动

从其他记忆插件迁移或首次安装时，可通过 `/memory coldstart [天数]` 命令扫描过去的历史对话并批量构建初始记忆。冷启动复用与实时对话相同的总结、主题匹配和片段合并逻辑，无需额外配置。

### 管理界面

提供 Web 管理界面，记忆对用户全透明。可以按自己的项目、兴趣点自由组织主题和记忆，形成对自己最有效的记忆结构。
